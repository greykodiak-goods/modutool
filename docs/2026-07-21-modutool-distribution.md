# 모두의툴 — 노출(유입) 전략 (2026-07-21)

> 전작 `2026-07-21-modutool-strategy.md`(제품·수익화)의 후속. 이 문서는 **트래픽을 어떻게 만들 것인가**만 다룬다.
> 전제: 신규 도메인은 구글 실질 유입까지 3~6개월 지연이 상수(리서치 검증됨). 따라서 노출 전략의 뼈대 = **①구글 SEO는 씨앗을 최대한 빨리·많이 심고 ②그 공백기를 비구글 채널(네이버·커뮤니티·다음/빙)로 버티고 ③계절 이벤트를 선점**하는 3축이다.

## 1. 채널별 우선순위 (기대 유입 × 통제 가능성)

| 순위 | 채널 | 역할 | 비고 |
|---|---|---|---|
| 1 | 구글 검색 (롱테일) | 본진 — 6개월 후 유입의 대부분 | 위젯 없는 키워드만 타깃 |
| 2 | 다음·빙 검색 | **저경쟁 조기 유입** — 등록 즉시 노출 가능, 경쟁자들이 무시하는 구간 | 다음은 네이트에도 동시 노출 |
| 3 | 네이버 (블로그 우회) | 외부 사이트는 통합검색 상단 진입이 구조적으로 막힘 → 블로그 글로 우회 | 효과 미검증, 소규모 실험 |
| 4 | 메이커/개발 커뮤니티 | 초기 백링크 + 직접 유입 + 색인 촉진 | 1회성 스파이크, 런칭 주에 집중 |
| 5 | AI 검색 (AI Overviews·챗봇 인용) | 성장 중인 무료 노출면 | 이미 구조 대응 완료(FAQ·정의문) |
| 6 | 크롬 확장·임베드 위젯 | 2차 로드맵 — 검색 무관 상시 유입·백링크 자석 | 안착 후 |

**하지 않는 것(금지선)**: 커뮤니티 도배·자문자답(디시·클리앙 등에 링크 뿌리기), 저품질 디렉토리 대량 등록(펭귄 페널티), 가짜 평점 마크업, "국내 최초/유일" 카피(사실 아님 — 검증됨), 트래픽 구매. 단기 스파이크보다 도메인 신뢰가 자산이다.

## 2. 구글 SEO — 키워드→랜딩 맵 (씨앗 심기)

원칙: **1키워드 = 1랜딩**(PDF24가 유사 페이지 2장으로 랭크됨을 실증), title에 연도("2026") + "서버 업로드 없음" 명시, 본문에 실예시(worked example) 필수.

**현재 13페이지가 커버하는 키워드** (이미 완료):
PDF 합치기/나누기/페이지 삭제/순서 바꾸기(→pdf-organize)/PDF JPG 변환/JPG PDF 변환(SERP 동일)/사진 PDF 만들기(→img-to-pdf), 이미지 압축/사진 용량 줄이기, 사진 크기 조절/WEBP 변환, 평수 계산기/평 제곱미터 변환, 디데이 계산기/100일 계산기, 글자수 세기/자소서 글자수, 퍼센트 계산기, 만나이(저기대)·삼각함수(저기대).

**확장 랜딩 25~35장 (스프린트 2, 기존 툴 재사용 — 개발비 거의 0)**:
- pdf-organize 쪼개기(4장): "PDF 회전", "PDF 페이지 추출", "PDF 페이지 삭제"(전용), "PDF 순서 바꾸기"(전용)
- 목표 KB 시나리오(8~12장): "토익 사진 500KB", "이력서 사진 용량 줄이기(100KB)", "정부24 제출 사진", "대학 지원 서류 PDF 10MB 이하", "사진 300KB 이하로", "여권/반명함 사진 규격(px 프리셋)" 등 — **접수처별 실제 규격을 표로 명시**(이 표가 스니펫 자산)
- 계산기 꼬리(4~6장): "수능 디데이"(전용, 9월 전 필수), "전역일 계산기", "근무일수 계산기", "84제곱미터 몇평"(콘텐츠형)
- 비교/보안 콘텐츠(8~15장, 블로그 섹션): "PDF 합치기 사이트 5곳 비교", "온라인 PDF 사이트에 회사 문서 올려도 되나"(수요 실증됨), 각 툴 사용법 튜토리얼 — 애드센스 승인 요건 겸용

**기술 체크(이미 반영됨)**: sitemap 자동생성, 전 페이지 정적+외부요청 0(CWV Good — 모바일 페이지 48%만 통과하는 시장에서 구조적 우위), FAQ 마크업(리치결과는 죽었지만 AI 인용 대비 유지), 갱신일 표기.

**색인 가속 루틴**: 배포 직후 GSC URL 검사로 전 페이지 개별 색인 요청(요청 시 24h 내 크롤 사례 다수) → 신규 랜딩 발행 때마다 즉시 요청. 빙은 IndexNow 설정(발행 즉시 푸시).

## 3. 네이버 — 정면 말고 우회

- **서치어드바이저 등록 + 사이트맵/RSS 제출**은 기본(반영 1~4주). 단, 외부 독립 사이트는 '웹사이트' 영역에만 노출 — 여기서 큰 유입을 기대하지 않는다(정직한 전제).
- **신규 네이버 블로그 1개 개설**(모두의툴 명의): "○○ 하는 법" 실사용 가이드 글에서 툴로 링크. 네이버 검색은 자사 UGC를 우대하므로 이게 합법적 우회로다. 효과는 미검증 — **주 1글 × 8주 소규모 실험 후 지표로 지속 판단**.
- ⚠️ 어닝하우스 기존 블로그·인스타 계정에 툴 홍보를 섞지 않는다(주제 불일치는 두 채널 모두에 감점). 단, **콘텐츠 팩토리(예약작업) 인프라 자체는 재사용** — sns_posts 파이프라인처럼 "모두의툴 블로그 글 초안 자동 생성" 트랙을 별도로 붙일 수 있다(원하면 내가 구축).
- 지식iN: 실제 질문에 실답변이 되는 경우만, 링크는 보조로(도배 금지). 월 수 건 수준.

## 9. 다국어 노출 (2026-07-21 추가 — 영어판 /en/ 구축 완료)

- **방식**: IP 국가 자동 리다이렉트는 SEO 안티패턴(구글봇이 미국에서 크롤 → 클로킹 오인 위험)이라 배제. 정석 구성 적용 완료 — 한국어=루트, 영어=`/en/…` 별도 URL + 전 페이지 hreflang(ko/en/x-default) + 브라우저 언어 감지 "제안 배너"(강제 이동 없음, 선택 저장).
- **기대치(정직하게)**: "merge pdf" 같은 영어 헤드는 한국어보다 훨씬 초경쟁이라 진입 기대 안 함. 영어판의 현실적 가치는 ① **한국 연관 키워드**("Korean age calculator", "pyeong to square meters" — 외국인 실수요, 경쟁 약함) ② 영어권 RPM이 한국보다 높아 유입당 수익 상방 ③ 글로벌 롱테일 낙수 ④ 추가 비용 ~0(로직 공유, 문자열만 번역). 
- **운영**: 영어판은 별도 콘텐츠 투자 없이 한국어판 갱신에 따라가는 "무임승차" 트랙으로 유지. GSC에서 en 지표가 유의미해지면 그때 영어 시나리오 랜딩 확장 검토. 제3언어(일본어 등)는 en 지표 확인 후.
- **광고 주의**: 애드핏은 한국 전용 — 승인 후 ads-config에 lang 분기 필요(en은 애드센스만).
