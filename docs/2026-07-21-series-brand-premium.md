# "thisismy" 시리즈 브랜딩 + 프리미엄/회원 시스템 설계 (2026-07-21)

> 대표 지시: ① thisismypdf·thisismyimg… 시리즈로 — 중복 검색 후 프로젝트·도메인 구조 설계 ② ilovepdf.com/pricing처럼 유료 범위 시스템 + 로그인·가입.

## 1. 네이밍 실사 결과 (⚠️ 대표 결정 필요)

- **"thisismypdf" 정확 일치 활성 사이트·브랜드는 검색에서 확인되지 않음** → 사용 가능으로 보임.
- **그러나 itsmypdf.com이 존재** — 이름이 한 끗 차이인데 포지셔닝까지 우리와 동일("browser-based, no upload, no sign-up, no watermark"). 그 외 likemypdf.com, fixmypdf.in, mypdfsite.com, whereismypdf.com 등 "~myPDF" 네임스페이스가 이미 붐빔.
- "thisismy" 단독은 Franz Enzenhofer의 크롬 확장(ChatGPT 브리핑 도구)이 사용 중 — 분야가 달라 차단 사유는 아님.
- 도메인 등록 여부 최종 확인은 프록시 차단으로 미완 — **구매 시점에 등록기관에서 확인 필수**.

**대표 선택지**
- A. **thisismypdf 강행**: 정확 일치 없음, 법적 리스크 낮음(설명적 조합이라 상표 약함 — 역으로 우리도 남을 막기 어려움). 단 itsmypdf와 혼동·오타 트래픽 유출 감수.
- B. 시리즈 콘셉트 유지 + 변형 검토 (예: thisismy.tools 우산 도메인 등) — 구매 전 재실사 1회.
- C. modutool 단일 브랜드 유지(현행) — 시리즈 감성은 포기, 혼동 리스크 0.

## 2. 시리즈/도메인 구조 (권장안)

**Phase 1 — 도메인 1개로 시작 (강력 권장)**: 시리즈 1호 = PDF 버티컬(가장 강한 클러스터). 이미지·계산기 도구도 당분간 같은 도메인의 서브경로로 운영.
- 근거: 신규 도메인은 유입까지 3~6개월 샌드박스가 상수 — 도메인을 3개로 쪼개면 이 대기와 콘텐츠·권위 축적이 3배로 분산된다. **iLovePDF도 ilovepdf.com을 키운 뒤에 iloveimg.com을 분사**했다(선례).

**Phase 2 — 트래픽 검증 후 분사**: 1호 도메인이 자리 잡으면(색인·유입 지표 기준) thisismyimg.com을 분사해 이미지 도구를 301 이관, 이어서 thisismycalc.com. 시리즈 간 상호 헤더 링크로 네트워크 효과.

**코드 구조**: 본 레포(modutool)가 시리즈 공용 코드베이스. 현 `__ORIGIN__` 주입 패턴을 확장해 분사 시 사이트 config(브랜드명·도메인·포함 툴 목록)로 멀티사이트 빌드 — 코드는 한 벌 유지(어닝옵스 "화면 한 벌" 원칙과 동일).

**브랜드명 반영 방법**: 현행 코드는 "모두의툴/ModuTool"로 중립 유지 중. 이름 확정 시 site.js LANGS + 페이지 title/JSON-LD 일괄 치환 1커밋이면 끝(구조 변경 없음).

## 3. 프리미엄 요금제 설계 (ilovepdf/pricing 벤치마크, 오늘 구현 완료)

**대원칙: 무료 기능은 절대 축소하지 않는다.** 트래픽이 자산인 성장기에 제한을 걸면 자멸(리서치 근거 — "무제한 무료"가 우리 콘텐츠 전체의 차별점). iLovePDF의 "무료 하루 2작업" 모델은 따라하지 않는다. 프리미엄은 **기본을 잠그는 게 아니라 편의를 얹는** 티어.

## 4. 구현된 회원 시스템 아키텍처

- **주입식 활성화(광고와 동일 패턴)**: `assets/auth-config.js`에 Supabase URL+anon key를 넣는 순간 전 사이트에서 로그인 버튼·회원 기능이 켜진다. 비어 있으면 흔적 없음(현재 상태 — 배포에 지장 없음).
- **성능 분리**: 전 페이지에는 3KB 경량 훅(`assets/auth.js`)만 — 세션 존재·플랜 캐시 확인, 헤더 버튼(프리미엄/로그인/내 계정), 프리미엄이면 광고 미로딩. Supabase SDK(204KB, `assets/vendor/supabase.js`)는 /login /signup /account 3페이지에서만 로드 → 툴 페이지 CWV 무영향.
- **페이지**: /pricing(색인 대상, ko·en) + /login /signup /account(noindex, ko·en). 로그인=비밀번호+매직링크, 가입=약관 동의+이메일 확인.
- **DB**: `supabase/migration_01_profiles.sql` — profiles(id→auth.users, plan free|premium, plan_expires_at), RLS 본인 조회만, **플랜 쓰기는 서버 전용**(클라이언트 자가승급 차단), 가입 트리거 자동 생성.
- **보안 노트**: 클라이언트 플랜 캐시는 광고 표시 편의 판정 전용. 돈이 걸린 실기능(배치·OCR 게이트)은 출시 때 서버 검증(Edge Function/RLS)으로 붙인다.

## 5. ⚠️ 백엔드 활성화 대기 — Supabase 슬롯 (대표 결정)

무료 플랜 상한(활성 프로젝트 2개: 어닝 운영 + 경매)에 걸려 신규 프로젝트 생성이 거부됨. 옵션:
1. **당분간 미활성(현 상태, 추천)** — 사이트 배포·SEO·광고에 지장 없음. 로그인 버튼은 자동 숨김. 트래픽이 생기기 전 회원 기능은 급하지 않음.
2. Supabase Pro 업그레이드($25/월) — 금전, 트래픽 검증 후 판단 권장.
3. 경매(realestate-auction) 프로젝트 일시정지 — 크롤러가 죽으므로 비추천.

활성화 절차(결정 후 5분): 프로젝트 생성 → migration_01 적용 → Auth 설정(이메일 확인, Site URL) → auth-config.js에 URL/키 기입 → 배포.
