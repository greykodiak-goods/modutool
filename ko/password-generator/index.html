비밀번호 생성기🔒 내 기기에서만 생성 — 전송 없음

브라우저의 암호학적 난수 생성기 crypto.getRandomValues로 내 기기에서 만드는 강력한 무작위 비밀번호. 어디에도 전송·기록·저장되지 않습니다 — 브라우저 저장소에도요.

문자 종류를 하나 이상 선택해야 비밀번호를 만들 수 있습니다.

사용 방법

  1. 위에 이미 비밀번호가 준비되어 있습니다 — 페이지가 열리는 순간 내 기기에서 생성된 것입니다.
  2. 길이와 문자 종류를 조절하면 비밀번호와 강도 측정이 즉시 다시 생성됩니다.
  3. 복사를 눌러 가입 폼이나 비밀번호 관리자에 붙여넣으세요. 다시 생성은 언제든 새로 뽑아 줍니다.

"내 기기에서 생성"이 핵심인 이유

서버에서 돌아가는 비밀번호 생성기는 그 자체로 모순입니다: 내가 가진 가장 비밀스러운 문자열이 남의 인프라와 로그에서 태어나는 셈이니까요. 이 도구는 모든 글자를 브라우저 안에서, 브라우저가 TLS 암호화에 쓰는 것과 같은 암호학적 난수 생성기 crypto.getRandomValues로 만듭니다. 비밀번호는 전송되지 않고, 로그에 남지 않고, 어떤 저장소에도 기록되지 않습니다 — 탭을 닫으면 사라집니다. 브라우저 네트워크 탭을 열면 아무 요청도 비밀번호를 싣고 나가지 않는 것을 볼 수 있습니다.

꼼꼼한 생성기와 대충 만든 생성기를 가르는 구현 디테일 두 가지:

  • 모듈로 편향 없음. 무작위 바이트를 % 연산으로 문자 풀에 대충 매핑하면 풀 앞쪽 문자가 미세하게 더 자주 나옵니다. 이 도구는 기각 샘플링 — 편향을 만드는 바이트 값은 버림 — 을 사용해 모든 문자가 정확히 같은 확률로 뽑힙니다.
  • 선택한 종류는 반드시 포함. 숫자나 특수문자가 빠진 비밀번호를 거부하는 사이트가 많습니다. 생성기는 선택한 각 문자 종류가 최소 1자씩 들어갔는지 확인하고, 아니면 무작위성을 해치지 않는 방식으로 새로 뽑습니다.

비밀번호는 몇 자가 적당할까

  • 16자(기본값) + 전체 문자 종류 = 약 104비트 엔트로피 — 온라인·오프라인 어느 쪽의 현실적인 무차별 대입도 불가능한 수준입니다.
  • 사이트가 짧은 길이를 강제한다면 12자가 합리적인 하한선입니다 — 전체 종류 기준 약 78비트.
  • 다른 모든 것을 지키는 비밀번호 — 비밀번호 관리자의 마스터 비밀번호, 디스크 암호화, 주 이메일 — 는 20자 이상으로.
  • 길이가 복잡성을 이깁니다: 특수문자 종류 하나 추가보다 4자 연장이 더 효과적입니다. 폼이 허용하면 길게 가세요.
💡 팁: 강도 측정은 무작위 생성 비밀번호에만 유효합니다. 사람이 지은 같은 길이의 비밀번호("여름2026!")는 공격자가 사람의 패턴부터 시도하기 때문에 측정치보다 훨씬 약합니다.

자주 묻는 질문

난수는 어떻게 만들어지나요?

crypto.getRandomValues — 모든 최신 브라우저에 내장된 암호학적으로 안전한 난수 생성기(CSPRNG)로, 암호화 키를 만들 때 쓰는 것과 같은 등급입니다. 전부 내 기기 안에서 실행됩니다. 여기에 더해 무작위 바이트를 문자로 바꿀 때 기각 샘플링을 사용해, 풀에 있는 모든 문자가 정확히 같은 확률로 뽑히도록(모듈로 편향 없음) 처리합니다.

헷갈리는 문자 제외는 왜 필요한가요?

0과 O, 1과 l과 I, | 같은 문자는 많은 글꼴에서 거의 구분이 안 됩니다. 화면에 뜬 비밀번호를 눈으로 읽고 손으로 입력해야 하는 상황 — TV, 공유기 설정, 전화로 불러줄 때 — 에서 이런 문자는 짜증나는 오타의 주범입니다. 헷갈리는 6개 문자를 빼면 풀이 약간 줄어(글자당 엔트로피 소수점 수준) 손으로 입력할 비밀번호라면 남는 장사입니다. 비밀번호 관리자에만 넣어둘 비밀번호라면 꺼도 됩니다.

엔트로피 비트가 무슨 뜻인가요?

공격자가 시도해야 하는 경우의 수를 나타내는 단위로, n비트면 2^n가지 가능성이 있다는 뜻입니다. 길이 × log2(문자 풀 크기)로 계산하며, 90자 풀에서 뽑은 16자 비밀번호는 약 104비트입니다. 대략 45비트는 작정한 오프라인 공격에 뚫리고, 70비트 안팎이면 대부분의 계정에 충분하며, 100비트 이상은 현실적인 무차별 대입 범위를 벗어납니다. 단, 이 계산은 무작위 생성 비밀번호에만 적용됩니다 — 사람이 지은 같은 길이의 비밀번호는 실질 엔트로피가 훨씬 낮습니다.

생성된 비밀번호가 어딘가에 저장되나요?

아니요. 비밀번호는 이 페이지의 메모리에만 존재하고, 페이지를 떠나거나 새로 생성하면 사라집니다. 서버로 전송되지 않고, localStorage나 쿠키에도 기록되지 않으며, 로그에도 남지 않습니다 — 익명 사용 통계에는 '몇 자짜리 비밀번호가 복사됐다'는 숫자만 남고 비밀번호 자체는 절대 포함되지 않습니다. 네트워크 탭에서 아무것도 나가지 않는 것을 직접 확인할 수 있습니다.

비밀번호 관리자를 쓰는 게 낫지 않나요?

솔직히 말하면 맞습니다 — 일상 계정에는 비밀번호 관리자가 더 나은 체계입니다. 계정마다 고유한 비밀번호를 생성·저장·자동입력해 재사용을 원천 차단하니까요. 이 도구는 관리자를 대체하는 게 아니라 보완합니다: 관리자가 채워줄 수 없는 비밀번호(와이파이 키, 기기 PIN, OS 계정, 관리자 자체의 마스터 비밀번호)나, 내 관리자가 없는 컴퓨터에서 일회성으로 강력한 비밀번호가 필요할 때 유용합니다.
