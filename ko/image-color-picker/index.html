이미지 색상 추출(스포이드)🔒 서버 업로드 없음

마우스를 올리면 커서 아래 색이 읽히고, 클릭하면 고정, 견본을 클릭하면 HEX 코드가 복사됩니다. 무료 — 설치·가입 없음.

🎨 이미지를 끌어다 놓거나 클릭해서 선택
한 번에 한 장 (JPG · PNG · WEBP · GIF · BMP)

이미지 위에서 움직이면 색이 읽힙니다 · 클릭하면 보관(최대 12개, 최신 순) · 견본을 클릭하면 HEX 복사.

사용 방법

  1. 이미지를 위 영역에 끌어다 놓거나 클릭해서 선택합니다 — 스크린샷, 사진, 로고 무엇이든 좋습니다.
  2. 미리보기 위에서 포인터를 움직이면 커서를 따라다니는 작은 칩에 그 픽셀의 HEX·RGB 값이 표시됩니다.
  3. 클릭하면 색이 고정되어 이미지 아래 견본 줄에 추가됩니다(최대 12개, 최신 순). 한 번에 팔레트 전체를 모을 수 있습니다.
  4. 아무 견본이나 — 또는 마지막 색을 복사하는 HEX 복사 버튼을 — 클릭하면 코드가 클립보드에 복사됩니다. CSS·피그마·브랜드 문서에 바로 붙여 넣으세요.

값이 의미하는 것

화면의 모든 픽셀은 빨강·초록·파랑 빛의 혼합이고, 각각 0~255 값을 가집니다. RGB 표기는 이를 그대로 나열하고 — rgb(37, 99, 235) — HEX는 같은 세 숫자를 16진수로 묶어 적습니다: #2563EB. 서로 완전히 호환되며, 대부분의 도구와 스타일 가이드가 HEX를 기대하기 때문에 복사 버튼도 HEX를 내보냅니다.

솔직히 짚어둘 것 하나: 브라우저 캔버스는 sRGB로 색을 읽습니다. 광색역(최신 폰에서 흔한 Display P3)으로 저장된 사진은 먼저 sRGB로 변환되므로, 채도가 아주 높은 색은 포토샵 같은 색 관리 편집기의 값과 몇 단계 차이 날 수 있습니다. 일반적인 웹 그래픽·스크린샷은 값이 정확히 일치합니다.

정확하게 집는 요령

  • 확대하기 — 브라우저 확대(Ctrl/Cmd +) 또는 모바일 핀치 줌. 스포이드가 확대된 이미지를 따라가므로 1픽셀 디테일도 쉽게 집힙니다.
  • 단색 영역에서 집기 — JPG 압축은 경계의 픽셀을 섞기 때문에, 글자·테두리 경계의 픽셀은 혼합값으로 읽힙니다. 단색 영역 한가운데가 진짜 값입니다.
  • PNG 스크린샷 선호 — PNG는 무손실이라 UI 색이 그대로 보존됩니다. JPG 스크린샷은 단색도 1~2 정도 틀어질 수 있습니다.
💡 팁: 사진에서 팔레트를 뽑을 때는 밝은 부분·중간톤·어두운 부분을 골고루 클릭해 두세요 — 견본 줄이 최근 12개를 보관하므로 비교하며 하나씩 복사할 수 있습니다.

이런 때 좋아요

  • 스타일 가이드를 찾을 수 없을 때 로고에서 브랜드 색상 맞추기
  • 스크린샷에서 본 배색 그대로 재현하기
  • 발표자료·썸네일에 쓸 포인트 색을 사진에서 뽑기
  • 실제 렌더링된 페이지가 어떤 색으로 나갔는지 확인
  • 디자인 시작 전에 빠르게 팔레트 수집

자주 묻는 질문

이미지가 어딘가로 업로드되나요?

아니요. 이미지는 브라우저 안의 캔버스에 그려지고, 색상은 그 캔버스에서 내 기기 위에서 읽습니다. 브라우저 개발자도구의 네트워크 탭을 열어보면 파일이 전송되지 않는 것을 직접 확인할 수 있고, 탭을 닫으면 모든 것이 사라집니다.

포토샵이나 디자인 파일과 HEX 값이 조금 다르게 나오는데요?

브라우저 캔버스는 sRGB로 동작합니다. 사진이 Display P3 같은 광색역(최신 아이폰에서 흔함)으로 저장돼 있으면 브라우저가 픽셀을 읽기 전에 sRGB로 변환하므로, 채도 높은 색은 색 관리를 하는 편집 프로그램의 값과 몇 단계 다를 수 있습니다. 원래 sRGB였던 픽셀 — 대부분의 웹 그래픽과 스크린샷 — 은 값이 정확히 일치합니다.

원하는 픽셀을 정확히 집으려면 어떻게 하나요?

확대하세요: PC에서는 브라우저 확대(Ctrl/Cmd +), 모바일에서는 손가락 핀치 줌을 쓰면 됩니다 — 읽기는 확대된 픽셀을 따라가므로 1픽셀짜리 디테일도 집을 수 있습니다. 클릭하기 전에 커서를 따라다니는 칩이 보여주는 색이 안정되는 것을 확인한 뒤 고정하세요.

HEX와 RGB는 뭐가 다른가요?

같은 색을 적는 두 가지 표기법입니다. RGB는 빨강·초록·파랑 성분을 0~255 숫자로 나열합니다(예: rgb(37, 99, 235)). HEX는 같은 세 숫자를 16진수 한 줄로 적습니다(#2563EB). CSS는 둘 다 인식하지만 디자인 도구·브랜드 가이드는 보통 HEX를 쓰기 때문에, 이 도구의 복사 버튼도 HEX를 복사합니다.

스크린샷에서 집은 색이 그 사이트의 CSS 값과 왜 다른가요?

JPG로 저장된 스크린샷은 경계 부분의 색이 살짝 섞입니다 — 압축이 이웃 픽셀을 섞기 때문에 글자나 버튼 테두리 경계의 픽셀은 혼합된 값으로 읽힙니다. 단색 영역의 한가운데에서 집고, 가능하면 PNG 스크린샷을 쓰세요. PNG는 무손실이라 단색 영역의 값이 그대로 보존됩니다.
