QR versions and error correction, briefly and honestly

A QR code grows in fixed steps called versions — version 1 is a 21×21 grid, and each version adds 4 modules per side up to version 40 (177×177). This tool picks the smallest version that fits your text automatically, so more text simply means a denser code.

Error correction reserves part of the code for redundant recovery data so a dirty, torn, or partly covered code still scans: L survives ~7% damage, M ~15%, Q ~25%, H ~30%. The trade-off is real — higher levels leave less room for your data and make the code denser. M is the sensible default. Choose H only when the code will be printed small, laminated onto equipment, or otherwise abused.

Wi-Fi QR codes: how guests use them

A Wi-Fi QR code encodes your network name, security type, and password in a standard format that iOS and Android both understand natively. A guest opens their camera, points it at the code, and taps the prompt — the phone joins the network without anyone spelling out "capital B, underscore, seven…". Print it and stick it on the fridge, frame it for your café counter, or drop it into a guest-room welcome card.

One honest caveat: the password sits in plain text inside the code. Anyone who scans it — including anyone with a photo of it — can read the password. Treat the printed code with the same care as the password itself.

Print size: how big should the code be?

  • Rule of thumb: scanning distance ≈ 10× the code's printed width. A 3 cm code scans comfortably from ~30 cm (a table tent); a poster read from 3 m needs a code around 30 cm wide.
  • Keep the quiet zone — the white margin around the code — intact. This tool bakes in the standard 4-module margin; don't crop it off.
  • More text = smaller squares at the same print size. If a dense code scans poorly, shorten the content (use a shorter URL) before enlarging the print.
  • For print, download at 1024 px so the code stays sharp when scaled.
💡 Tip: Always test-scan a printed code with a real phone before producing 500 flyers. It takes ten seconds and catches every sizing mistake.

Frequently asked questions

Does a QR code made here ever expire?

No. A QR code is nothing more than your text drawn as a pattern of squares — there is no server, no account, and no link-shortening service behind it. As long as the content itself stays valid (for example, the URL still works), the code will scan forever. Nothing is hosted anywhere that could be shut down.

Is it safe to put my Wi-Fi password into this tool?

The tool itself never transmits or stores anything — the QR code is generated entirely inside your browser, and you can verify in the Network tab that nothing is sent. Be aware of what a Wi-Fi QR code is, though: the password is encoded in plain text inside the pattern, so anyone who scans the code (or decodes a photo of it) can read the password. Share the printed code only with people you would tell the password to.

How much text fits in one QR code?

At the lowest error-correction level (L), a QR code holds up to about 2,953 bytes of data — roughly 2,900 Latin characters, fewer for Korean or emoji since they take 3 bytes each in UTF-8. Higher error-correction levels reserve more space for recovery data, so the limit drops — level H holds roughly 1,270 bytes. If your text does not fit, the tool tells you instead of producing a broken code.

Why does higher error correction make the code look denser?

Error correction adds redundant recovery data so the code still scans when partly dirty, damaged, or covered — L tolerates about 7% damage, M 15%, Q 25%, and H 30%. That extra data has to go somewhere, so the code needs more modules (smaller squares) for the same text. Use M for general use, and Q or H only for codes that will be printed small or handled roughly.

Can I put a logo in the middle of the QR code?

Not in this tool — it generates clean standard codes only. Logo overlays work by deliberately covering part of the code and relying on high error correction to survive, which is possible but fragile to get right. If you need a branded code, generate a high-error-correction (H) PNG here and add the logo carefully in an image editor, keeping it under about 20% of the code area, then test-scan it.
