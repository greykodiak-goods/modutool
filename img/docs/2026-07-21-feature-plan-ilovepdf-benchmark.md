# ThisIsMyPDF 기능 기획 — iLovePDF 전 기능 벤치마크 (2026-07-21)

> 대표 지시: "모든 기능 다 제대로 기획해서 만들어. 고객 니즈에 맞게. ilovepdf 참고."
> 원칙: 우리는 100% 클라이언트 사이드(무업로드)가 정체성이다 — iLovePDF 기능 중 **브라우저에서 제대로 되는 것은 전부 만들고, 서버 없이는 품질이 안 나오는 것은 정직하게 로드맵/보류로 구분**한다. 흉내만 내는 저품질 구현은 브랜드에 독이라 하지 않는다.

## iLovePDF 기능 전수 대비표

| iLovePDF 기능 | 고객 니즈(검색 수요) | 우리 상태 | 판단 근거 |
|---|---|---|---|
| Merge PDF | PDF 합치기 — 상위 수요 | ✅ 있음 (/pdf-merge) | |

| Organize PDF | 페이지 삭제·순서 | ✅ 있음 (/pdf-organize) | 리서치 1순위 |
| Rotate PDF | PDF 회전 | ✅ 있음 (/pdf-rotate) | |
| PDF → JPG | 변환 | ✅ 있음 (/pdf-to-jpg) | |
| JPG → PDF | 변환 | ✅ 있음 (/img-to-pdf) | |
| **Watermark** | 대외비·초안 표시 | 🆕 **오늘 구축** (/pdf-watermark) | pdf-lib + 한글 폰트 임베드(Noto KR 서브셋 2.4MB, 실행 시점 로드) |
| **Page numbers** | 쪽번호 (제출 규정) | 🆕 **오늘 구축** (/pdf-page-numbers) | pdf-lib. 한글 형식은 폰트 임베드 |
| **Sign PDF** | 계약서 사인 회신 | 🆕 **오늘 구축** (/pdf-sign) | 사인 패드→PNG 삽입. "인증서 전자서명 아님" 정직 고지 |
| Edit PDF (텍스트/도형) | 간단 주석 | 🔜 로드맵 2차 | 텍스트 상자·도형 삽입은 pdf-lib로 가능 — 서명 툴 코드 재사용해 확장 |
| Crop PDF | 여백 자르기 | 🔜 로드맵 2차 | pdf-lib CropBox + 드래그 UI |
| Protect (암호 걸기) | 암호 설정 | 🔜 로드맵 3차 | pdf-lib 미지원 → qpdf-wasm 등 WASM 검토 필요 |
| Unlock PDF | 암호 해제 | ⏸ 보류 | 기술 가능해도 권한 우회 오남용 소지 — 정책 판단 필요 |
| OCR PDF | 스캔→텍스트 | 🔜 **프리미엄 로드맵**(가격표에 예고됨) | tesseract.js(클라이언트 가능, 한글 데이터 ~15MB) — 유료 차별 기능 적합 |
| PDF ↔ Word/Excel/PPT | 오피스 변환 — 수요 큼 | ⏸ 보류(정직) | 클라이언트 사이드로는 레이아웃 품질이 안 나옴. 저품질 출시는 브랜드 손상 — 서버/WASM(LibreOffice) 대규모 검토 후에만 |
| PDF/A, Repair, Compare, Redact | 니치 | ⏸ 보류 | 수요 대비 구현비 과다 |
| Scan to PDF | 모바일 스캔 | 🔜 로드맵 3차 | 카메라 API + 보정. img-to-pdf 확장 |
| HTML → PDF | 웹페이지 저장 | ⏸ 보류 | 브라우저 인쇄 기능과 중복 |

오늘 이후 PDF 스위트: **11개 툴** (합치기·나누기·추출·페이지편집·회전·압축·워터마크·쪽번호·서명·PDF→JPG·이미지→PDF) — iLovePDF 무료 핵심 세트에서 오피스 변환·암호 계열만 남는 수준.

## iLovePDF에서 채택한 것 / 의도적으로 다르게 간 것

**채택**: 1기능=1페이지 구조, 홈 그리드, 기능별 명확한 CTA, Premium 티어(광고 제거+고급 기능), 가격 페이지 구조.
**다르게**: ①무료 제한 없음(그들은 일 2작업 제한 — 우리는 무업로드라 서버비가 없어 제한할 이유가 없고, 이게 콘텐츠 전체의 차별점) ②파일이 서버로 가지 않음 ③저품질 기능은 안 만듦(오피스 변환 등은 보류를 명시).

## 고객 니즈 → 페이지 설계 반영 (전 툴 공통 템플릿)

- 니즈 1위 "빨리 끝내기": 열자마자 드롭존, 가입·설치 0, 3클릭 내 결과
- 니즈 2위 "내 파일 안전한가": 전 페이지 "서버 업로드 없음" 배지 + 처리 방식 설명 + 프라이버시 정책
- 니즈 3위 "제출 조건 맞추기": 목표 KB 압축(이미지), 용량 제한·제출 규정 사용 사례를 가이드에 명시
- 실패 시 친절한 한국어 안내(암호 PDF·손상 파일), 원본 비파괴(항상 새 파일 다운로드)
