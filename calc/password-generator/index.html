Password Generator🔒 Local only — never transmitted

Strong random passwords generated on your device with crypto.getRandomValues, the browser's cryptographic RNG. Nothing is transmitted, logged, or stored — not even in your browser's local storage.

Select at least one character class to generate a password.

How to use

  1. A password is already waiting above — it was generated the moment the page loaded, on your device.
  2. Adjust the length and character classes; the password and its strength meter regenerate instantly with every change.
  3. Click Copy and paste it into the sign-up form or your password manager. Regenerate rolls a fresh one any time.

Why "generated locally" is the whole point

A password generator that runs on a server is a contradiction: the most secret string you own would begin its life in someone else's infrastructure and logs. This tool generates every character inside your browser using crypto.getRandomValues, the same cryptographically secure generator the browser uses for TLS. The password is never transmitted, never logged, and never written to any storage — close the tab and it is gone. Open your browser's Network tab and you will see that no request carries it anywhere.

Two implementation details worth knowing, because they separate careful generators from careless ones:

  • No modulo bias. Naively mapping random bytes onto a character pool with % makes early characters in the pool slightly more likely. This tool uses rejection sampling — biased byte values are discarded — so every character is exactly equally probable.
  • Every selected class appears. Many sites reject passwords missing a digit or symbol. The generator verifies at least one character from each class you selected and re-rolls if needed, without weakening randomness.

How long should a password be?

  • 16 characters (the default) with all classes on gives ~104 bits of entropy — beyond any realistic brute-force attack, online or offline.
  • 12 characters is a reasonable floor when a site enforces a shorter limit — still ~78 bits with all classes.
  • 20+ characters for things that guard everything else: your password manager's master password, disk encryption, your primary email.
  • Length beats complexity: adding 4 characters helps more than adding one more symbol class. When a form allows it, go longer.
💡 Tip: The strength meter measures randomly generated passwords only. A human-made password of the same length ("Summer2026!") is dramatically weaker than the meter would suggest, because attackers try human patterns first.

Frequently asked questions

How is the randomness generated?

With crypto.getRandomValues — the cryptographically secure random number generator (CSPRNG) built into every modern browser, the same class of generator used for encryption keys. It runs entirely on your device. The tool additionally uses rejection sampling when mapping random bytes to characters, so every character in the pool is exactly equally likely (no modulo bias).

Why exclude look-alike characters?

Characters like 0/O, 1/l/I, and | are nearly indistinguishable in many fonts. If you ever have to read a password off a screen or type it by hand — on a TV, a router label, or over the phone — look-alikes cause maddening errors. Excluding the six look-alike characters shrinks the pool slightly (a fraction of a bit of entropy per character), which is a worthwhile trade for anything typed by hand. For passwords that only live in a password manager, feel free to turn the option off.

What do the entropy bits mean?

Entropy measures how many guesses an attacker would need: n bits means 2^n equally likely possibilities. It is computed as length × log2(pool size), so a 16-character password from a 90-character pool has about 104 bits. As a rough guide, 45 bits falls to a determined offline attack, around 70 bits is solid for most accounts, and 100+ bits is beyond any realistic brute force. Entropy only applies to random passwords — a human-invented password of the same length has far fewer effective bits.

Is the generated password stored anywhere?

No. It exists only in this page's memory and disappears when you leave or generate a new one. It is not sent to any server, not written to localStorage or cookies, and not logged — our anonymous usage counter records only that a password of a certain length was copied, never the password itself. You can verify in the Network tab that nothing leaves your device.

Should I use a password manager instead?

Honestly: yes, for day-to-day accounts a password manager is the better system — it generates, stores, and fills unique passwords so you never reuse one. This tool complements a manager rather than replacing it: it is handy for the passwords a manager cannot fill (Wi-Fi keys, device PINs, an OS account, the manager's own master password) or when you need a one-off strong password on a machine without your manager.
