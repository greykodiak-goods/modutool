텍스트 비교🔒 서버 전송·저장 없음

두 버전의 글을 붙여넣으면 무엇이 바뀌었는지 한눈에 — 추가는 초록, 삭제는 빨강, 줄 단위 또는 단어 단위. 비교는 전부 브라우저 안에서 이뤄지고 텍스트는 전송되지 않습니다.

양쪽에 텍스트를 붙여넣으세요 — 입력하는 동안에도 자동으로 비교됩니다.

    

  


  


  

    
사용 방법

    
    
      
  1. 이전 버전을 원본에, 새 버전을 비교본에 붙여넣습니다.
    2. 
      
  2. 줄 단위 또는 단어 단위를 고릅니다. 입력하는 동안 자동으로 결과가 갱신되고, 비교하기를 눌러도 됩니다.
    3. 
      
  3. 하이라이트를 읽습니다: 초록 = 추가, 빨강 취소선 = 삭제. 요약 줄에 바뀐 조각 개수가 표시됩니다.
    4. 
    


    
줄 단위와 단어 단위, 언제 무엇을

    
    
      
  • 줄 단위 — 줄 하나하나가 의미 단위인 텍스트에 적합: 코드 조각, 설정 파일, CSV 행, 목록. 줄 안에 하나라도 다르면 그 줄 전체가 바뀐 것으로 표시되므로, 줄바꿈이 크게 달라진 산문은 문단째 표시됩니다.
    • 
      
  • 단어 단위 — 산문에 적합: 자소서, 계약서, 이메일. 문장 안에서 바뀐 단어만 짚어 주므로, 한 단어 수정이 문단 전체가 아니라 한 단어로 보입니다.
    • 
    

    
경험칙: 줄바꿈에 의미가 있으면 줄 단위부터, 문장이 흘러가는 글이면 단어 단위. 모드를 바꾸면 즉시 다시 비교되니 둘 다 눌러 봐도 손해가 없습니다.


    
공백도 차이로 칩니다 — 의도된 동작

    
단어 단위 모드는 공백까지 민감하게 비교합니다: 한 칸이 두  칸이 되거나 스페이스가 탭으로 바뀌면 변경으로 보고합니다. 코드·서식 있는 텍스트·데이터처럼 간격이 중요한 문서를 확인할 때는 보통 이게 원하는 동작입니다. 눈으로는 안 보이는 "변경"이 표시되면 간격과 보이지 않는 문자를 의심하세요 — 줄 끝의 공백도 차이로 집계됩니다.


    
이런 때 좋아요

    
    
      
  • 계약서 수정본 — 상대방이 v3와 v4 사이에 정확히 어떤 조항을 고쳤는지, 상대의 요약을 믿는 대신 직접 확인.
    • 
      
  • 코드 조각 — 버전 관리에 없는 함수·설정 두 벌을 빠르게 대조.
    • 
      
  • 과제·논문 — 초고와 첨삭본 사이에 무엇이 바뀌었는지, 혹은 내 퇴고가 어디를 건드렸는지 점검.
    • 
      
  • 번역·사본 대조 — 출처가 다른 "같은" 텍스트(약관, 상품 설명, 메일 템플릿)가 정말 일치하는지 확인.
    • 
    

    
💡 팁: 아무것도 업로드되지 않으므로 기밀 자료에 써도 됩니다 — 랜선을 뽑아도 비교는 그대로 동작합니다.


    
자주 묻는 질문

    

      
비교할 때 텍스트가 어딘가로 업로드되나요?
아니요. 비교 알고리즘은 브라우저 안에서 자바스크립트로 실행됩니다 — 두 텍스트 모두 내 기기에 머물고, 아무것도 전송·저장되지 않습니다. 브라우저 개발자도구의 네트워크 탭에서 텍스트를 싣고 나가는 요청이 없다는 것을 직접 확인할 수 있습니다. 그래서 계약서, 미공개 원고 같은 기밀 자료에도 안심하고 쓸 수 있습니다.

      
텍스트 크기 제한이 있나요?
정해진 제한은 없습니다 — 실질적인 한계는 브라우저의 메모리와 CPU입니다. 1MB 안팎(수백 페이지 분량)까지는 무리 없이 비교되고, 그보다 훨씬 크면 조각을 더 많이 살피는 단어 단위 모드부터 느려질 수 있습니다. 큰 비교가 버벅이면 줄 단위 모드로 바꿔 보세요 — 훨씬 잘 버팁니다.

      
글자 단위로도 비교할 수 있나요?
이 도구에서는 안 됩니다 — 단어 단위가 제공되는 가장 세밀한 모드입니다. 실제로 문서·산문에는 단어 단위 하이라이트가 정답에 가깝습니다. 보통 글에 글자 단위 결과를 적용하면 색종이 조각처럼 잘게 흩어져 오히려 읽기 어려워집니다. 일련번호 비교처럼 글자 수준 정밀도가 정말 필요한 경우에도, 단어 모드가 달라진 토큰을 따로 짚어 줍니다.

      
초록·빨강 표시는 각각 무슨 뜻인가요?
초록은 비교본에는 있지만 원본에는 없는 텍스트 — 추가된 부분입니다. 빨강 취소선은 원본에는 있지만 비교본에서 삭제되거나 대체된 텍스트 — 삭제된 부분입니다. 교체된 곳은 빨강 조각 바로 옆에 초록 조각이 붙어 나타납니다. 표시가 없는 텍스트는 두 버전에서 동일한 부분입니다.

      
파일 두 개를 직접 비교할 수 있나요?
이 버전에는 파일 선택 기능이 없습니다 — 각 파일을 열어 전체 선택 후 두 입력창에 붙여넣으세요. .txt·.md·.csv·소스코드·메모 내보내기 등 일반 텍스트 파일이면 무엇이든 됩니다. .docx나 .pdf 같은 바이너리 형식은 해당 프로그램에서 텍스트를 복사해 붙여넣어야 합니다.
