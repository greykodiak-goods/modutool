QR코드 생성기🔒 서버 전송·저장 없음

링크·텍스트·와이파이 접속 정보를 QR코드로 — 실시간 미리보기, 선명한 PNG 저장. 전부 브라우저 안에서 만들어지며 입력한 내용은 전송·저장되지 않습니다.

위에 내용을 입력하면 여기에 QR코드가 나타납니다.

사용 방법

  1. 내용 종류를 고릅니다: 링크·전화번호·메시지는 URL / 텍스트, 손님이 스캔해서 바로 접속하게 하려면 와이파이 접속.
  2. 내용을 입력하면 미리보기가 실시간으로 갱신됩니다 — 따로 누를 버튼이 없습니다.
  3. 필요하면 크기(저장될 PNG의 픽셀)와 오류 복원을 조절하세요. 기본값(512px, M 레벨)이면 대부분 충분합니다.
  4. PNG 다운로드를 누르면 qr-code.png로 저장되고, 이미지 복사를 누르면 채팅·문서에 바로 붙여넣을 수 있습니다.

QR 버전과 오류 복원, 짧고 정직하게

QR코드는 버전이라는 정해진 단계로 커집니다 — 버전 1이 21×21 격자이고, 버전이 하나 오를 때마다 한 변에 4모듈씩 늘어 버전 40(177×177)까지 있습니다. 이 도구는 입력한 내용이 들어가는 가장 작은 버전을 자동으로 고르므로, 내용이 길수록 코드가 빽빽해질 뿐입니다.

오류 복원은 코드 일부가 더러워지거나 가려져도 인식되도록 복구용 데이터를 미리 넣어 두는 기능입니다: L은 약 7%, M은 15%, Q는 25%, H는 30% 손상까지 버팁니다. 대신 레벨이 높을수록 내용이 들어갈 자리가 줄고 코드가 빽빽해지는 실제 트레이드오프가 있습니다. 일반 용도는 M이 정답이고, 작게 인쇄하거나 장비에 붙여 험하게 쓸 코드에만 H를 고르세요.

와이파이 QR코드: 손님은 이렇게 씁니다

와이파이 QR코드는 네트워크 이름·보안 방식·비밀번호를 iOS와 Android가 모두 기본 지원하는 표준 형식으로 담습니다. 손님이 카메라를 켜고 코드를 비춘 뒤 안내를 탭하면 끝 — "대문자 B에 언더바, 숫자 7…" 하고 불러줄 필요가 없습니다. 인쇄해서 냉장고에 붙이거나, 카페 카운터에 액자로 세우거나, 에어비앤비 안내 카드에 넣어 두면 좋습니다.

정직한 주의 하나: 비밀번호는 코드 안에 평문으로 들어갑니다. 코드를 스캔한 사람 — 코드 사진을 가진 사람 포함 — 은 누구나 비밀번호를 읽을 수 있으니, 인쇄물은 비밀번호 자체와 같은 수준으로 관리하세요.

인쇄 크기: 코드는 얼마나 크게?

  • 경험칙: 스캔 거리 ≈ 코드 인쇄 폭의 10배. 3cm 코드는 약 30cm 거리(테이블 안내)에서 편하게 찍히고, 3m 떨어져 보는 포스터라면 30cm짜리 코드가 필요합니다.
  • 코드 둘레의 흰 여백(콰이어트 존)을 살려 두세요. 이 도구는 표준 4모듈 여백을 포함해 저장합니다 — 잘라내지 마세요.
  • 내용이 길수록 같은 인쇄 크기에서 사각형이 작아집니다. 빽빽한 코드가 잘 안 찍히면 인쇄를 키우기 전에 내용부터 줄이세요(짧은 URL 사용).
  • 인쇄용은 1024px로 받아야 확대해도 선명합니다.
💡 팁: 전단지 500장을 뽑기 전에 인쇄본을 실제 폰으로 꼭 스캔해 보세요. 10초면 크기 실수를 전부 잡아냅니다.

자주 묻는 질문

여기서 만든 QR코드에 유효기간이 있나요?

없습니다. QR코드는 입력한 텍스트를 사각형 패턴으로 그려 놓은 것일 뿐, 뒤에 서버·계정·단축링크 서비스가 없습니다. 내용 자체가 유효한 한(예: URL이 살아 있는 한) 코드는 영원히 인식됩니다. 어딘가에 호스팅되어 있다가 중단될 것이 아예 없기 때문입니다.

와이파이 비밀번호를 입력해도 안전한가요?

이 도구 자체는 아무것도 전송하거나 저장하지 않습니다 — QR코드는 전부 브라우저 안에서 생성되며, 네트워크 탭에서 아무 요청도 나가지 않는 것을 직접 확인할 수 있습니다. 다만 와이파이 QR코드의 성격은 알아두세요: 비밀번호가 패턴 안에 평문으로 인코딩되므로 코드를 스캔한 사람(코드 사진을 가진 사람 포함)은 누구나 비밀번호를 읽을 수 있습니다. 인쇄한 코드는 비밀번호를 알려줘도 되는 사람에게만 보여주세요.

QR코드 하나에 글자를 얼마나 담을 수 있나요?

오류 복원 레벨이 가장 낮은 L 기준 최대 약 2,953바이트입니다 — 영문·숫자로 약 2,900자, 한글은 UTF-8에서 한 글자에 3바이트라 약 980자 정도입니다. 복원 레벨을 올리면 복구용 데이터가 자리를 차지해 한도가 줄어들고, H 레벨은 약 1,270바이트입니다. 내용이 넘치면 깨진 코드를 만드는 대신 안내 메시지를 보여줍니다.

오류 복원 레벨을 올리면 왜 코드가 더 빽빽해지나요?

오류 복원은 코드가 일부 더러워지거나 찢기거나 가려져도 인식되도록 복구용 데이터를 추가하는 기능입니다 — L은 약 7%, M은 15%, Q는 25%, H는 30%의 손상까지 버팁니다. 그 추가 데이터가 들어갈 자리가 필요하므로 같은 내용이라도 모듈(사각형)이 많아져 더 빽빽해집니다. 일반 용도는 M이면 충분하고, 작게 인쇄하거나 험하게 다뤄질 코드에만 Q·H를 쓰세요.

QR코드 가운데에 로고를 넣을 수 있나요?

이 도구에서는 안 됩니다 — 표준 형태의 깨끗한 코드만 생성합니다. 로고 삽입은 코드 일부를 일부러 가리고 높은 오류 복원력으로 버티게 하는 방식이라 가능은 하지만 잘못하면 인식이 안 되기 쉽습니다. 브랜드 코드가 필요하면 여기서 복원 레벨 H로 PNG를 만든 뒤 이미지 편집기에서 코드 면적의 20% 이하로 로고를 얹고, 반드시 실제 폰으로 스캔 테스트를 하세요.
