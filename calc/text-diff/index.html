Text Diff🔒 No upload, no storage

Paste two versions of a text and see exactly what changed — additions in green, deletions in red, line by line or word by word. Compared entirely in your browser; your text is never transmitted.

Paste text into both boxes — the comparison also runs automatically as you type.

  1. Paste the earlier version into Original and the newer version into Changed.

Line mode or word mode?

    
    
      
  • Lines — best for structured text where lines are meaningful units: code snippets, config files, CSV rows, lists. A line counts as changed if anything in it differs, so heavily re-wrapped prose lights up whole paragraphs.
    • 
      
  • Words — best for prose: essays, contracts, emails. It pinpoints the exact words that changed inside a sentence, so a one-word edit shows as one word, not a whole paragraph.
    • 
    

    
Rule of thumb: if the text has meaningful line breaks, start with line mode; if it's flowing sentences, use word mode. Switching modes re-runs the comparison instantly, so trying both costs nothing.


    
Whitespace counts — deliberately

    
Word mode uses a whitespace-sensitive comparison: changing one space to one  two spaces, or a space to a tab, is reported as a change. That is usually what you want when checking documents where spacing matters (code, formatted text, data). Keep it in mind when a diff shows "changes" you can't see at first glance — look for spacing and invisible characters, and note that trailing spaces at line ends count too.

Frequently asked questions

    

      
Is my text uploaded anywhere when I compare?
No. The comparison algorithm runs as JavaScript inside your browser — both texts stay on your device, and nothing is transmitted or stored. You can confirm in the Network tab of your browser's developer tools that no request carries your text. That makes it safe for contracts, unpublished drafts, and anything else confidential.

      
Is there a size limit on the texts?
There is no hard limit — the practical ceiling is your browser's memory and CPU. Texts up to around 1 MB (hundreds of pages) compare comfortably; far beyond that, word-level mode in particular can get slow because it examines many more pieces. If a huge comparison feels sluggish, switch to line mode, which scales much better.

      
Can it compare character by character?
Not in this tool — word mode is the finest granularity offered. In practice word-level highlighting is what you want for prose and documents; character-level output on ordinary text tends to produce confetti that is harder to read, not easier. If you truly need character-level precision (say, comparing serial numbers), word mode will still isolate the differing token for you.

      
What do the green and red highlights mean?
Green marks text that exists in the changed version but not the original — additions. Red with strikethrough marks text that exists in the original but was removed or replaced in the changed version — deletions. A replacement shows up as a red part immediately next to a green part. Unhighlighted text is identical in both versions.

      
Can I compare two files directly?
This version has no file picker — open each file, select all, and paste the contents into the two boxes. That works for any plain-text file: .txt, .md, .csv, source code, exported notes. Binary formats like .docx or .pdf need their text extracted first (copy out of the application and paste here).
