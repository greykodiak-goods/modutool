What makes this counter different

Plenty of sites will count characters for you, but few say what happens to the text you paste in. A personal statement or college essay carries your name, school, and work history — sensitive information by definition — and if a counter processes text on its server, you have no way of knowing where that text ends up.

The ModuTool counter sends your text nowhere. Counting happens entirely inside the browser you are looking at; nothing is written to a server or even to browser storage, and closing the page erases the text completely. Open the network tab in your browser's developer tools and you will see that no request carries your input anywhere.

Character limits: with or without spaces?

When an application form says "within 1,000 characters," whether that includes spaces varies by company and application system — the same employer's own careers page and a job platform can even disagree with each other. This matters most for personal statements and essays submitted to Korean and other Asian application systems, which usually limit by characters rather than by words. When in doubt:

  1. Look for an explicit "including spaces" or "excluding spaces" note in the posting or next to the input field — it is usually there in parentheses.
  2. If there is no note, paste a short test sentence with spaces into the actual form and watch which way the remaining-character counter moves.
  3. Still unsure? Write to the stricter with-spaces limit. That prevents the classic accident of the form silently cutting off the end of your essay at submission.

Some systems — notably Korean public-sector application forms — limit by bytes instead of characters, and the byte count depends on the encoding. In UTF-8, the current web standard, one Korean character is 3 bytes (letters, digits, and spaces are 1 byte each); in EUC-KR, an older Korean 2-byte encoding still used by some legacy systems, one Korean character is 2 bytes. This tool shows both side by side, so read whichever standard your submission system uses.

What are "manuscript pages"?

The manuscript-page figure is based on wongoji — traditional Korean manuscript paper, a grid of 200 squares with one character per square, still widely used as a length unit in Korean writing contests, school essays, and publishing. This tool divides the with-spaces character count by 200 and rounds up. For example, 1,530 characters with spaces gives 1,530 ÷ 200 = 7.65 → 8 pages.

Treat it as an estimate. On real manuscript paper, each paragraph starts with a blank square, an early line break wastes the rest of that row, and punctuation takes a square of its own — so the true page count usually comes out a little higher. If a contest says "about 10 pages," leave yourself a margin of one or two pages.

Frequently asked questions

Do character limits for personal statements include spaces?

It depends on the company or application system. First check whether the instructions say 'including spaces' or 'excluding spaces'; if nothing is stated, it is safest to stay within the stricter with-spaces count. This tool shows both counts side by side, so you can check against either standard instantly.

Is my text sent to or stored on a server?

No. Everything is counted inside your browser, and your text is never transmitted or saved anywhere. Closing or refreshing the page erases it completely. It is safe to paste personal statements or essays that contain private information.

Why do byte counts differ between sites?

Because they assume different encodings. In UTF-8, the current web standard, one Korean character is 3 bytes; in EUC-KR, an older Korean 2-byte encoding still used by some legacy systems, one Korean character is 2 bytes. This tool shows both counts, so read the one that matches the system you are submitting to.

How are manuscript pages calculated?

It is a quick estimate based on the traditional Korean 200-character manuscript sheet: the with-spaces character count divided by 200, rounded up. Copying text onto real manuscript paper adds blank squares for paragraph indents and line breaks, so the actual page count can come out higher — leave a margin of one or two pages.

Do line breaks count as characters?

In the with-spaces count, each line break counts as one character; in the without-spaces count it is removed along with all other whitespace. Most web forms count the same way, but the submission form itself is always the final authority — check there when it matters.
